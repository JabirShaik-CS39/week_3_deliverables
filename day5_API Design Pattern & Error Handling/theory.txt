1. Search Endpoints
A search endpoint allows users to find resources based on keywords or criteria.

# Example

Imagine an e-commerce application with products.

Endpoint
GET /api/v1/products/search?q=laptop
Response
{
    "products": [
        {
            "id": 1,
            "name": "Dell Laptop",
            "price": 50000
        }
    ]
}
FastAPI Example
from fastapi import FastAPI, Query

app = FastAPI()

products = [
    {"id": 1, "name": "Dell Laptop"},
    {"id": 2, "name": "HP Laptop"},
    {"id": 3, "name": "iPhone"}
]

@app.get("/products/search")
def search_products(q: str = Query(...)):
    result = [
        product for product in products
        if q.lower() in product["name"].lower()
    ]
    return result

## Standardized Error Responses
All API errors should follow the same structure.

Bad Example
{
  "error": "User not found"
}

Another endpoint:

{
  "message": "Invalid email"
}

Inconsistent.

Good Standard Format
{
  "success": false,
  "error": {
      "code": "USER_NOT_FOUND",
      "message": "User does not exist"
  }
}


Validation Error Format
{
    "success": false,
    "error": {
        "code": "VALIDATION_ERROR",
        "message": "Email is required",
        "field": "email"
    }
}


Internal Server Error
{
    "success": false,
    "error": {
        "code": "SERVER_ERROR",
        "message": "Something went wrong"
    }
}

Benefits
--> Easy frontend integration
--> Consistent responses
--> Easier debugging
--> Better documentation


## Custom Exception Handlers

Instead of writing try-except everywhere, define global handlers.

Custom Exception
class ProductNotFoundException(Exception):
    def __init__(self, product_id):
        self.product_id = product_id
Exception Handler
from fastapi import Request
from fastapi.responses import JSONResponse

@app.exception_handler(ProductNotFoundException)
async def product_exception_handler(
    request: Request,
    exc: ProductNotFoundException
):
    return JSONResponse(
        status_code=404,
        content={
            "success": False,
            "error": {
                "code": "PRODUCT_NOT_FOUND",
                "message": f"Product {exc.product_id} not found"
            }
        }
    )
Usage
@app.get("/products/{id}")
def get_product(id: int):

    product = find_product(id)

    if not product:
        raise ProductNotFoundException(id)

    return product
Advantages
Centralized error handling
Cleaner code
Reusable


## HTTPException Usage

FastAPI provides HTTPException.

Syntax
from fastapi import HTTPException
Example
@app.get("/users/{id}")
def get_user(id: int):

    user = find_user(id)

    if not user:
        raise HTTPException(
            status_code=404,
            detail="User not found"
        )

    return user

## Request ID Middleware

Every request gets a unique ID.

Useful for tracing logs.

Example

Request:

GET /products

Generated:

request_id=abc123
Middleware
import uuid
from fastapi import Request

@app.middleware("http")
async def add_request_id(
    request: Request,
    call_next
):
    request_id = str(uuid.uuid4())

    request.state.request_id = request_id

    response = await call_next(request)

    response.headers["X-Request-ID"] = request_id

    return response
Logging Request ID
logger.info(
    "request_received",
    request_id=request.state.request_id
)
Benefits

Imagine:

10000 requests/day

A user reports an issue.

You can search logs:

request_id=abc123

and find the complete request flow.


## API Versioning

Versioning prevents breaking existing clients.

Bad
/api/products

If API changes, old applications break.

Good
/api/v1/products
/api/v2/products
FastAPI Example
from fastapi import APIRouter

v1_router = APIRouter(prefix="/api/v1")

@v1_router.get("/products")
def get_products():
    return {"version": "v1"}
Version 2
v2_router = APIRouter(prefix="/api/v2")

@v2_router.get("/products")
def get_products():
    return {
        "version": "v2",
        "currency": "INR"
    }
Benefits
Backward compatibility
Safe upgrades
Easier maintenance
